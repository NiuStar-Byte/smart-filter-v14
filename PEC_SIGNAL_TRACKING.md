# PEC Signal Tracking - Complete Guide

## Problem We Solved

**Before:** PEC was backtesting on ALL signals in `signals_fired.jsonl` (34K+), including:
- Signals that were generated but rejected by RR filter
- Signals that failed to reach Telegram
- Signals that never actually executed

**Result:** PEC results didn't match reality (phantom trades)

---

## Solution: SENT_SIGNALS.jsonl

Now we track **ONLY signals that actually reached Telegram** in a separate file:

```
SENT_SIGNALS.jsonl
├── Entry: Symbol, Timeframe, Entry Price, TP, SL
├── Score: Filter score, Confidence, Route
├── Telegram: Message ID (proof it was sent)
├── Status: OPEN, TP_HIT, SL_HIT, TIMEOUT
├── Execution: Exit price, P&L (when closed)
└── Tracking: fired_time_utc, sent_time_utc, closed_at
```

---

## File Structure

### signals_fired.jsonl (OLD - for archiving)
- ALL signals generated by SmartFilter
- May include rejections
- NOT used by PEC anymore

### SENT_SIGNALS.jsonl (NEW - PEC uses this)
- ONLY signals sent to Telegram
- With complete execution tracking
- Ready for PEC to validate and track

## Workflow: How Signals Flow to PEC

```
1. SmartFilter generates signal
   ↓
2. Passes RR filter (MIN_ACCEPTED_RR)
   ↓
3. Stored to signals_fired.jsonl (archive)
   ↓
4. Sent to Telegram
   ↓
5. ✅ LOGGED TO SENT_SIGNALS.jsonl ← PEC USES THIS
   ↓
6. Trader executes on Telegram alert
   ↓
7. PEC tracks: TP hit? SL hit? Timeout?
   ↓
8. Update SENT_SIGNALS with exit_price, pnl_usd, pnl_pct
```

## Integration: What PEC Should Do

### Step 1: Read SENT_SIGNALS (not signals_fired)
```python
import json

def get_pec_signals():
    """Get signals ready for PEC execution tracking"""
    signals = []
    with open('SENT_SIGNALS.jsonl', 'r') as f:
        for line in f:
            signals.append(json.loads(line))
    return signals
```
