### 1. P&L Calculation Bug Fix (CRITICAL) — 18:09–18:25
**Commit:** 9d3369f (submodule) + a5d5827 (workspace)

**Issue:** pec_executor.py used current_price instead of target prices for TP/SL P&L calculations
```
WRONG (before):
  TP_HIT @ $135 (Entry $100, Target $131) → P&L = +$35 ❌
  SL_HIT @ $88 (Entry $100, Target $90.69) → P&L = -$12 ❌

CORRECT (after):
  TP_HIT → P&L = (131-100)/100 × notional = +$31 ✓
  SL_HIT → P&L = (100-90.69)/100 × notional = -$9.31 ✓
```

**Impact:** RR ratio validation now passes exactly (Avg TP / Avg SL = Target RR)
- Example: If RR = 1.31, then Avg TP = 1.31 × Avg SL (confirmed post-fix)

**Fix Applied:**
- TP_HIT: Uses tp_target price (recognized target)
- SL_HIT: Uses sl_target price (recognized target)
- TIMEOUT: Uses current_price (target never reached — correct behavior)

**Note:** Treated as bug fix (internal logic), NOT new feature. 5 locked trackers code-unchanged, results auto-correct.
